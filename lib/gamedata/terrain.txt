# Archivo: terrain.txt


# Este archivo se utiliza para inicializar la información de las "características del terreno" para
# el juego Angband.

# No modifiques este archivo a menos que sepas exactamente lo que estás haciendo,
# a menos que quieras arriesgarte a posibles fallos del sistema y archivos guardados corruptos.

# Ten en cuenta que, a diferencia de otros archivos editables, modificar este archivo
# de forma casual probablemente sea una MALA IDEA (a menos que tengas la intención de hacer cambios
# reales en la programación) por dos razones:

#   1. Las características del terreno no aparecen a menos que haya código específico en los
#   algoritmos de generación de niveles para incluirlas, por lo que añadir nuevas características
#   del terreno a este archivo por sí solo no tendrá ningún efecto.

#   2. Cambiar las propiedades (especialmente las banderas, pero también el mimetismo, la excavación,
#   y la bandera de resistencia) puede cambiar significativamente el comportamiento del juego, y
#   cualquier cambio probablemente estropeará una partida que se inició antes de que los cambios
#   tuvieran lugar y se recargue cuando los cambios estén en vigor.

# Puedes alterar más libremente los gráficos, nombres, mensajes y descripciones.
# Hay un vínculo implícito entre los símbolos de las tiendas y las teclas de acceso directo
# en el menú principal de conocimiento para mostrar el contenido de una tienda (esas teclas
# se establecen por el orden de las tiendas dentro de list-terrain.h). Ten en cuenta que la
# característica de terreno cero contiene la imagen de "oscuridad".

# === Comprendiendo terrain.txt ===

# code : nombre del código de list-terrain.h
# name : nombre imprimible del terreno
# graphics : símbolo : color
# mimic : nombre del código de la característica a imitar
# priority : prioridad de visualización en el minimapa
# flags : banderas del terreno
# digging : índice de excavación
# walk-msg: mensaje de advertencia - caminar
# run-msg: mensaje de advertencia - correr
# hurt-msg: mensaje de daño
# die-msg: mensaje de muerte
# confused-msg: mensaje para cuando un monstruo confundido intenta moverse a un terreno no transitable
# look-prefix: se usa antes del nombre en el resultado al mirar
# look-in-preposition: preposición usada cuando se mira la propia posición
# resist-flag: bandera de raza de monstruo para la resistencia

# 'code' indica el inicio de una entrada. El valor utilizado para el código
# debe coincidir con el argumento de una de las macros FEAT() en list-terrain.h.

# 'name' especifica el nombre imprimible del terreno. Como se muestra
# en la barra de estado y varios menús, es preferible un nombre corto.

# 'graphics' es para los gráficos - símbolo y color. Hay 27 colores:

# D - Gris Oscuro    w - Blanco          s - Gris          o - Naranja
# r - Rojo          g - Verde           b - Azul          u - Ocre
# W - Gris Claro    P - Púrpura Claro   y - Amarillo      R - Rojo Claro
# G - Verde Claro   B - Azul Claro      U - Ocre Claro    p - Púrpura
# v - Violeta       t - Verde Azulado   m - Barro         Y - Amarillo Claro
# i - Magenta-Rosa  T - Verde Azulado Claro V - Violeta Claro I - Rosa Claro
# M - Mostaza       z - Pizarra Azul    Z - Azul Claro Profundo

# 'mimic' es el nombre del código (uno de los argumentos de las macros FEAT() en
# list-terrain.h) de una característica a imitar; algunas características parecen
# otra característica hasta que el jugador las descubre.

# 'priority' es para la prioridad de visualización en el minimapa, donde solo se
# selecciona un subconjunto de casillas. Esto permite que el minimapa muestre la
# información más probable de ser útil para el jugador.

# 'flags' son las banderas del terreno.

# 'digging' es para especificar la dificultad de excavar a través del terreno. Esto
# es opcional y, si el terreno, según sus banderas, no es excavable, establecer
# el índice no tiene un efecto sustancial más allá de la comprobación de errores
# durante el análisis. La dificultad se especifica como un número entero del uno
# al cinco (es decir, uno más una de las constantes DIGGING_* de player.h) donde
# uno es como los escombros, dos es como una veta de magma, tres es como una veta
# de cuarzo, cuatro es como el granito, y cinco es como una puerta cerrada.

# 'walk-msg' es el mensaje de advertencia para el jugador al pisar un terreno dañino

# 'run-msg' es el mensaje de advertencia para el jugador al correr hacia un terreno dañino

# 'hurt-msg' es el mensaje que recibe el jugador cuando el terreno le causa daño

# 'die-msg' es la descripción si el terreno dañino mata al jugador

# 'confused-msg' es la descripción del intento de un monstruo de moverse a un terreno
# no transitable debido a la confusión

# 'look-prefix' se imprime antes del nombre en el resultado de mirar el
# terreno. Si no se establece, el valor por defecto es usar 'un' si el nombre
# empieza por vocal o 'un' si el nombre no empieza por vocal.

# 'look-in-preposition' es la preposición que se usará al imprimir
# el resultado para un observador que mira su propia casilla: 'Estás
# <preposición> <prefijo> <nombre>'. Si no se establece, el valor por defecto
# es usar 'en'.

# 'resist-flag' es la bandera de raza que debe tener un monstruo para poder entrar
# en un terreno dañino

# 'desc' es para la descripción. Se pueden usar tantas líneas 'desc:' como sean necesarias
# para describir el terreno. Ten en cuenta que las líneas necesitarán espacios al final
# o al principio de la siguiente línea para evitar que las palabras se junten.
