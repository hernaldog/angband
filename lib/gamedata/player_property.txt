# Archivo: player_property.txt

# Utilizado en conjunto con list-player-flags.h, list-object-flags.h y
# list-elements.h
# Propiedades del jugador

# Las descripciones y los nombres se pueden editar sin malos efectos.
# Cambiar el código o el tipo alterará gravemente el juego, y solo debe
# hacerse cuando los archivos list-*.h asociados (y otro código) también
# se estén cambiando, y el juego sea recompilado.
# Añadir nuevas entradas no tendrá ningún efecto sin más cambios en los archivos
# de datos y el código.

# Campos:
# type - el tipo de propiedad del jugador. Los tipos válidos son
#    player  - bandera especial de jugador de list-player-flags.h
#    object  - bandera de objeto de list-object-flags.h
#    element - resistencia, inmunidad o vulnerabilidad a un elemento
# code - el nombre de la propiedad utilizado por el juego. Los códigos de elemento son plantillas, y
#        se crea una para cada elemento
# bindui - Vincula la propiedad a un elemento de la interfaz de usuario en ui_entry.txt.
#          Al vincular una propiedad de elemento, el nombre se parametrizará
#          con el nombre del elemento. Toma dos parámetros. El primero es un
#          entero, que si es distinto de cero, indica que el valor se pasa al elemento
#          de la interfaz de usuario como auxiliar (usado en algunos para indicar un
#          sostenimiento, por ejemplo). El segundo es el valor a pasar. Puede ser
#          un entero o "special", donde esto último indica que el valor pasado
#          se determinará internamente. bindui actualmente solo afecta a partes de la
#          segunda pantalla de personaje y a la comparación de equipables. Puede aparecer
#          varias veces para la misma entrada para vincularla a múltiples elementos de
#          la interfaz de usuario.
# name - el nombre de la propiedad
# desc - la descripción de la propiedad
# value - valor para elementos; -1 (vulnerabilidad), 1 (resistencia) o 3 (inmunidad)
